  <h2>Prerequisites &amp; Environment Setup</h2>
  <p>Everything you need before writing a single line of project code. Set this up on Day 1 morning. Your laptop is the only server you need.</p>

  <h3>Hardware requirements</h3>
  <table>
    <thead><tr><th>Tier</th><th>Machine</th><th>Model choice</th><th>RAM needed</th></tr></thead>
    <tbody>
      <tr><td><span class="pill green">Best</span></td><td>MacBook Pro M2/M3 / RTX 4070+</td><td>gemma4:e4b (9.6 GB)</td><td>16 GB+</td></tr>
      <tr><td><span class="pill blue">Good</span></td><td>Any 16 GB laptop with CPU</td><td>gemma4:e4b (Q4 quantized)</td><td>12 GB+</td></tr>
      <tr><td><span class="pill amber">Minimum</span></td><td>8 GB RAM machine</td><td>gemma4:e2b (5.3 GB)</td><td>8 GB</td></tr>
    </tbody>
  </table>

  <div class="callout warning">
    <span class="callout-icon">⚠</span>
    <div class="callout-text">The E4B model is strongly recommended over E2B for this project. Its improved reasoning handles the multi-step triage logic, JSON extraction from messy handwriting images, and multi-tool orchestration far more reliably. Use E2B only if you hit real memory limits.</div>
  </div>

  <h3>Step 1 — Install Ollama</h3>
  <div class="code-block">
    <div class="code-header"><span class="code-lang">bash</span><span class="code-file">terminal</span></div>
    <pre><span class="cmt"># macOS / Linux</span>
curl -fsSL https://ollama.com/install.sh | sh

<span class="cmt"># Windows (WSL2 recommended)</span>
<span class="cmt"># Download installer from https://ollama.com/download</span>

<span class="cmt"># Verify Ollama is running</span>
ollama --version
<span class="cmt"># → ollama version 0.6.x</span>

<span class="cmt"># Start the Ollama server (if not auto-started)</span>
ollama serve</pre>
  </div>

  <h3>Step 2 — Pull the Gemma 4 E4B model</h3>
  <div class="code-block">
    <div class="code-header"><span class="code-lang">bash</span><span class="code-file">terminal</span></div>
    <pre><span class="cmt"># Pull Gemma 4 Edge 4B (the recommended model for this project)</span>

<span class="kw">import</span> json
<span class="kw">from</span> pathlib <span class="kw">import</span> Path

<span class="cmt"># ──────────────────────────────────────────────────────────────────────────
# Gemma 4 Visual Token Budgets
# Lower = faster (use for rough intake scans)
# Higher = more detail (use for handwritten forms, small text)
# Supported: 70, 140, 280, 560, 1120

TOKEN_BUDGETS = {

<span class="kw">def</span> <span class="fn">encode_image</span>(image_path: str) -> str:
    <span class="str">"""Convert local image file to base64 string for Gemma 4 vision."""</span>
    <span class="kw">with</span> open(image_path, <span class="str">"rb"</span>) <span class="kw">as</span> f:
        <span class="kw">return</span> base64.b64encode(f.read()).decode(<span class="str">"utf-8"</span>)

<span class="kw">def</span> <span class="fn">chat_text</span>(
    prompt: str,
    system: str = <span class="str">""</span>,
    temperature: float = <span class="num">1.0</span>,  <span class="cmt"># Google's recommended default for Gemma 4</span>
    json_mode: bool = <span class="kw">False</span>,
) -> str:
    <span class="str">"""Simple text-only Gemma 4 call."""</span>
    messages = []
    <span class="kw">if</span> system:
        messages.append({<span class="str">"role"</span>: <span class="str">"system"</span>, <span class="str">"content"</span>: system})
    messages.append({<span class="str">"role"</span>: <span class="str">"user"</span>, <span class="str">"content"</span>: prompt})

    kwargs = {<span class="str">"model"</span>: settings.GEMMA_MODEL, <span class="str">"messages"</span>: messages,
              <span class="str">"options"</span>: {<span class="str">"temperature"</span>: temperature}}
    <span class="kw">if</span> json_mode:
        kwargs[<span class="str">"format"</span>] = <span class="str">"json"</span>

    response = ollama.chat(**kwargs)
    <span class="kw">return</span> response[<span class="str">"message"</span>][<span class="str">"content"</span>]

<span class="kw">def</span> <span class="fn">chat_vision</span>(
    prompt: str,
    image_path: str,
    system: str = <span class="str">""</span>,
    detail: str = <span class="str">"medium"</span>,   <span class="cmt"># "low" | "medium" | "high"</span>
    json_mode: bool = <span class="kw">False</span>,
) -> str:
    <span class="str">"""
    Gemma 4 multimodal call — image + text prompt.
    The visual_token_budget option controls how much detail Gemma 4 
    allocates to the image. Use "high" for handwritten forms.
    """</span>
    img_b64 = encode_image(image_path)
    messages = []
    <span class="kw">if</span> system:
        messages.append({<span class="str">"role"</span>: <span class="str">"system"</span>, <span class="str">"content"</span>: system})
    messages.append({
        <span class="str">"role"</span>: <span class="str">"user"</span>,
        <span class="str">"content"</span>: prompt,
        <span class="str">"images"</span>: [img_b64],
    })

    kwargs = {
        <span class="str">"model"</span>: settings.GEMMA_MODEL,
        <span class="str">"messages"</span>: messages,
        <span class="str">"options"</span>: {
            <span class="str">"temperature"</span>: <span class="num">1.0</span>,
            <span class="str">"visual_token_budget"</span>: TOKEN_BUDGETS[detail],  <span class="cmt"># Gemma 4 specific!</span>
        },
    }
    <span class="kw">if</span> json_mode:
        kwargs[<span class="str">"format"</span>] = <span class="str">"json"</span>

    response = ollama.chat(**kwargs)
    <span class="kw">return</span> response[<span class="str">"message"</span>][<span class="str">"content"</span>]

<span class="kw">def</span> <span class="fn">chat_with_tools</span>(
    messages: list[dict],
    tools: list[dict],
    system: str = <span class="str">""</span>,
) -> dict:
    <span class="str">"""
    Gemma 4 native function calling.
    Returns the raw Ollama response — caller must check for tool_calls.
    
    IMPORTANT: The two-pass pattern is required:
    Pass 1 → Model returns tool_calls JSON
    Pass 2 → Execute tools, send results back, model returns final answer
    """</span>
    full_messages = []
    <span class="kw">if</span> system:
        full_messages.append({<span class="str">"role"</span>: <span class="str">"system"</span>, <span class="str">"content"</span>: system})
    full_messages.extend(messages)

    response = ollama.chat(
        model=settings.GEMMA_MODEL,
        messages=full_messages,
        tools=tools,
        options={<span class="str">"temperature"</span>: <span class="num">1.0</span>},
    )
    <span class="kw">return</span> response[<span class="str">"message"</span>]</pre>

  <h3>RAG implementation</h3>
  <div class="code-block">
    <div class="code-header"><span class="code-lang">python</span><span class="code-file">backend/ai/rag.py</span></div>
    <pre><span class="kw">from</span> langchain_community.document_loaders <span class="kw">import</span> PyPDFLoader, TextLoader
<span class="kw">from</span> langchain.text_splitter <span class="kw">import</span> RecursiveCharacterTextSplitter
<span class="kw">from</span> langchain_community.vectorstores <span class="kw">import</span> Chroma
<span class="kw">from</span> langchain_ollama <span class="kw">import</span> OllamaEmbeddings
<span class="kw">from</span> pathlib <span class="kw">import</span> Path
<span class="kw">from</span> config <span class="kw">import</span> settings

<span class="cmt"># ── EMBEDDING SETUP ────────────────────────────────────────────────────</span>
<span class="kw">def</span> <span class="fn">get_embeddings</span>():
    <span class="str">"""Local embeddings via Ollama — no internet required."""</span>
    <span class="kw">return</span> OllamaEmbeddings(model=settings.EMBED_MODEL)

<span class="cmt"># ── INDEX CORPUS ───────────────────────────────────────────────────────</span>
<span class="kw">def</span> <span class="fn">build_index</span>() -> Chroma:
    <span class="str">"""
    Load all PDFs and TXT files from the docs directory,
    chunk them, embed with nomic-embed-text, and store in ChromaDB.
    Run once — subsequent loads use the persisted index.
    """</span>
    docs_dir = Path(settings.DOCS_DIR)
    all_docs = []

    <span class="kw">for</span> path <span class="kw">in</span> docs_dir.iterdir():
        <span class="kw">if</span> path.suffix == <span class="str">".pdf"</span>:
            loader = PyPDFLoader(str(path))
        <span class="kw">elif</span> path.suffix <span class="kw">in</span> (<span class="str">".txt"</span>, <span class="str">".md"</span>):
            loader = TextLoader(str(path))
        <span class="kw">else</span>:
            <span class="kw">continue</span>
        all_docs.extend(loader.load())

    splitter = RecursiveCharacterTextSplitter(
        chunk_size=<span class="num">500</span>,   <span class="cmt"># 500 tokens per chunk — good for policy docs</span>
        chunk_overlap=<span class="num">80</span>,  <span class="cmt"># Overlap to avoid context splits mid-sentence</span>
    )
    chunks = splitter.split_documents(all_docs)

    vector_store = Chroma.from_documents(
        documents=chunks,
        embedding=get_embeddings(),
        persist_directory=settings.CHROMA_PERSIST_DIR,
    )
    print(f<span class="str">"✓ Indexed {len(chunks)} chunks from {len(all_docs)} documents"</span>)
    <span class="kw">return</span> vector_store

<span class="kw">def</span> <span class="fn">load_index</span>() -> Chroma:
    <span class="str">"""Load a previously built ChromaDB index. Instant on subsequent runs."""</span>
    <span class="kw">return</span> Chroma(
        persist_directory=settings.CHROMA_PERSIST_DIR,
        embedding_function=get_embeddings(),
    )

<span class="cmt"># ── RETRIEVAL ──────────────────────────────────────────────────────────</span>
<span class="kw">def</span> <span class="fn">retrieve</span>(
    query: str,
    vector_store: Chroma,
    k: int = <span class="num">4</span>,
) -> list[dict]:
    <span class="str">"""
    Retrieve top-k most relevant policy/SOP chunks for a given query.
    Returns a list of dicts with 'content' and 'source' keys.
    """</span>
    results = vector_store.similarity_search(query, k=k)
    <span class="kw">return</span> [
        {<span class="str">"content"</span>: doc.page_content, <span class="str">"source"</span>: doc.metadata.get(<span class="str">"source"</span>, <span class="str">"Unknown"</span>)}
        <span class="kw">for</span> doc <span class="kw">in</span> results
    ]

<span class="cmt"># ── GROUNDED GENERATION ────────────────────────────────────────────────</span>
<span class="kw">def</span> <span class="fn">generate_grounded_plan</span>(
    intake_record: dict,
    retrieved_docs: list[dict],
    gemma_fn,  <span class="cmt"># Pass in chat_text function</span>
) -> str:
    <span class="str">"""
    Given an extracted intake record and retrieved policy chunks,
    ask Gemma 4 to generate an action plan grounded in the documents.
    """</span>
    context = <span class="str">"\n\n"</span>.join([
        f<span class="str">"[{doc['source']}]\n{doc['content']}"</span>
        <span class="kw">for</span> doc <span class="kw">in</span> retrieved_docs
    ])
    prompt = f<span class="str">"""You are a disaster relief case manager.

INTAKE RECORD:
{intake_record}

RELEVANT RELIEF POLICIES AND STANDARDS:
{context}

Based ONLY on the intake record and the above policy documents, generate:
1. TRIAGE SUMMARY: 2-3 sentences describing the person's situation
2. IMMEDIATE ACTIONS: Numbered list of next steps (max 5)
3. RESOURCE NEEDS: Specific resources required
4. MISSING INFO: What the volunteer should ask next
5. ESCALATION: Whether this case needs supervisor review (YES/NO + reason)
6. POLICY CITATIONS: Which document sections justify your triage decision

Be specific, brief, and actionable. Real lives depend on this being clear."""</span>
    <span class="kw">return</span> gemma_fn(prompt)</pre>

  <h2>Function Calling Agent</h2>
  <p>This is the demo's biggest "WOW moment" — Gemma 4 calling real local tools in sequence, not just generating text. Use the two-pass Ollama pattern.</p>

  <h3>Tool definitions</h3>
  <p>Define your tool schemas as Python dicts that map to the Ollama tool-calling format:</p>

  <div class="tool-card">
    <div class="tool-header"><span class="tool-name">search_local_resources</span></div>
    <div class="tool-desc">Searches the local resource directory JSON for matching shelters, food banks, medical posts, or transport options.</div>
  </div>
  <div class="tool-card">
    <div class="tool-header"><span class="tool-name">create_case</span></div>
    <div class="tool-desc">Saves the extracted intake record to SQLite with a unique case ID and timestamp.</div>
  </div>
  <div class="tool-card">
    <div class="tool-header"><span class="tool-name">generate_referral_pdf</span></div>
    <div class="tool-desc">Produces a printable PDF referral packet with all case data, action plan, and resource addresses.</div>
  </div>

  <div class="code-block">
    <div class="code-header"><span class="code-lang">python</span><span class="code-file">backend/ai/agent.py</span></div>
    <pre><span class="kw">import</span> json
<span class="kw">import</span> ollama
<span class="kw">from</span> config <span class="kw">import</span> settings
<span class="kw">from</span> tools.resource_lookup <span class="kw">import</span> search_resources
<span class="kw">from</span> tools.case_manager <span class="kw">import</span> create_case_db, update_case_db
<span class="kw">from</span> tools.pdf_export <span class="kw">import</span> generate_pdf

<span class="cmt"># ── TOOL SCHEMAS (Ollama-compatible format) ────────────────────────────</span>
TOOLS = [
    {
        <span class="str">"type"</span>: <span class="str">"function"</span>,
        <span class="str">"function"</span>: {
            <span class="str">"name"</span>: <span class="str">"search_local_resources"</span>,
            <span class="str">"description"</span>: <span class="str">"Search local directory for shelters, food, medical, or transport resources near a location."</span>,
            <span class="str">"parameters"</span>: {
                <span class="str">"type"</span>: <span class="str">"object"</span>,
                <span class="str">"properties"</span>: {
                    <span class="str">"resource_type"</span>: {
                        <span class="str">"type"</span>: <span class="str">"string"</span>,
                        <span class="str">"enum"</span>: [<span class="str">"shelter"</span>, <span class="str">"food"</span>, <span class="str">"medical"</span>, <span class="str">"transport"</span>],
                        <span class="str">"description"</span>: <span class="str">"Type of resource needed"</span>,
                    },
                    <span class="str">"location"</span>: {
                        <span class="str">"type"</span>: <span class="str">"string"</span>,
                        <span class="str">"description"</span>: <span class="str">"Area or neighborhood to search near"</span>,
                    },
                    <span class="str">"family_size"</span>: {
                        <span class="str">"type"</span>: <span class="str">"integer"</span>,
                        <span class="str">"description"</span>: <span class="str">"Number of people needing the resource"</span>,
                    },
                },
                <span class="str">"required"</span>: [<span class="str">"resource_type"</span>],
            },
        },
    },
    {
        <span class="str">"type"</span>: <span class="str">"function"</span>,
        <span class="str">"function"</span>: {
            <span class="str">"name"</span>: <span class="str">"create_case"</span>,
            <span class="str">"description"</span>: <span class="str">"Save a new intake case to the local database and return a case ID."</span>,
            <span class="str">"parameters"</span>: {
                <span class="str">"type"</span>: <span class="str">"object"</span>,
                <span class="str">"properties"</span>: {
                    <span class="str">"intake_data"</span>: {<span class="str">"type"</span>: <span class="str">"object"</span>, <span class="str">"description"</span>: <span class="str">"The full extracted intake record"</span>},
                    <span class="str">"triage_level"</span>: {<span class="str">"type"</span>: <span class="str">"string"</span>, <span class="str">"enum"</span>: [<span class="str">"GREEN"</span>, <span class="str">"YELLOW"</span>, <span class="str">"ORANGE"</span>, <span class="str">"RED"</span>]},
                },
                <span class="str">"required"</span>: [<span class="str">"intake_data"</span>, <span class="str">"triage_level"</span>],
            },
        },
    },
]

<span class="cmt"># ── TOOL EXECUTOR ──────────────────────────────────────────────────────</span>
<span class="kw">def</span> <span class="fn">execute_tool</span>(tool_name: str, tool_args: dict) -> str:
    <span class="str">"""Route tool calls from Gemma 4 to the actual Python functions."""</span>
    <span class="kw">if</span> tool_name == <span class="str">"search_local_resources"</span>:
        results = search_resources(**tool_args)
        <span class="kw">return</span> json.dumps(results)
    <span class="kw">elif</span> tool_name == <span class="str">"create_case"</span>:
        case_id = create_case_db(**tool_args)
        <span class="kw">return</span> json.dumps({<span class="str">"case_id"</span>: case_id, <span class="str">"status"</span>: <span class="str">"created"</span>})
    <span class="kw">else</span>:
        <span class="kw">return</span> json.dumps({<span class="str">"error"</span>: f<span class="str">"Unknown tool: {tool_name}"</span>})

<span class="cmt"># ── MAIN AGENT LOOP ────────────────────────────────────────────────────</span>
<span class="kw">async def</span> <span class="fn">run_intake_agent</span>(intake_record: dict, rag_context: str) -> dict:
    <span class="str">"""
    Two-pass Gemma 4 agent loop:
    Pass 1: Model receives intake + context, decides which tools to call
    Pass 2: Tool results returned, model generates final action plan
    
    Returns: { case_id, triage_level, action_plan, resources, evidence }
    """</span>
    system = <span class="str">"""You are a disaster relief intake agent. 
You receive a structured intake record and policy context.
Your job is to:
1. Determine triage level (GREEN/YELLOW/ORANGE/RED)
2. Call search_local_resources for each needed resource type
3. Call create_case to save the record with the triage level
4. Generate a clear action plan for the relief volunteer

Always use tools when appropriate. Be decisive and brief."""</span>

    user_message = f<span class="str">"""INTAKE RECORD:
{json.dumps(intake_record, indent=2)}

POLICY CONTEXT:
{rag_context}

Process this case now."""</span>

    messages = [{<span class="str">"role"</span>: <span class="str">"user"</span>, <span class="str">"content"</span>: user_message}]

    <span class="cmt"># ── PASS 1: Let Gemma 4 decide what tools to call ──────────────────</span>
    response = ollama.chat(
        model=settings.GEMMA_MODEL,
        messages=[{<span class="str">"role"</span>: <span class="str">"system"</span>, <span class="str">"content"</span>: system}] + messages,
        tools=TOOLS,
        options={<span class="str">"temperature"</span>: <span class="num">1.0</span>},
    )
    assistant_message = response[<span class="str">"message"</span>]
    messages.append(assistant_message)

    <span class="cmt"># ── Execute each tool call Gemma requested ─────────────────────────</span>
    tool_results = {}
    <span class="kw">if</span> assistant_message.get(<span class="str">"tool_calls"</span>):
        <span class="kw">for</span> tool_call <span class="kw">in</span> assistant_message[<span class="str">"tool_calls"</span>]:
            fn_name = tool_call[<span class="str">"function"</span>][<span class="str">"name"</span>]
            fn_args = tool_call[<span class="str">"function"</span>][<span class="str">"arguments"</span>]
            result = execute_tool(fn_name, fn_args)
            tool_results[fn_name] = json.loads(result)
            messages.append({
                <span class="str">"role"</span>: <span class="str">"tool"</span>,
                <span class="str">"content"</span>: result,
            })

    <span class="cmt"># ── PASS 2: Final synthesis with tool results ──────────────────────</span>
    final_response = ollama.chat(
        model=settings.GEMMA_MODEL,
        messages=[{<span class="str">"role"</span>: <span class="str">"system"</span>, <span class="str">"content"</span>: system}] + messages,
        options={<span class="str">"temperature"</span>: <span class="num">1.0</span>},
    )
    action_plan = final_response[<span class="str">"message"</span>][<span class="str">"content"</span>]

    <span class="kw">return</span> {
        <span class="str">"case_id"</span>: tool_results.get(<span class="str">"create_case"</span>, {}).get(<span class="str">"case_id"</span>),
        <span class="str">"action_plan"</span>: action_plan,
        <span class="str">"resources_found"</span>: tool_results.get(<span class="str">"search_local_resources"</span>, {}),
        <span class="str">"tool_calls_made"</span>: list(tool_results.keys()),  <span class="cmt"># For transparency UI</span>
    }</pre>

<span class="cmt"># ── MAIN INTAKE ENDPOINT ───────────────────────────────────────────────</span>
<span class="kw">@app.post</span>(<span class="str">"/intake"</span>)
<span class="kw">async def</span> <span class="fn">process_intake</span>(
    image: UploadFile = File(<span class="kw">None</span>),       <span class="cmt"># Optional form photo</span>
    voice_text: str = Form(<span class="str">""</span>),           <span class="cmt"># Optional voice transcript</span>
    manual_text: str = Form(<span class="str">""</span>),          <span class="cmt"># Optional typed notes</span>

    Accepts: form image + voice transcript + manual text
    Returns: extracted record, triage level, action plan, resources, evidence
    """</span>
    intake_record = <span class="kw">None</span>

    <span class="cmt"># Step 1: Extract structured data from input</span>
    <span class="kw">if</span> image <span class="kw">and</span> image.filename:

        <span class="kw">async with</span> aiofiles.open(tmp_path, <span class="str">"wb"</span>) <span class="kw">as</span> f:

        <span class="kw">for</span> field, val <span class="kw">in</span> voice_data.dict().items():
            <span class="kw">if</span> val <span class="kw">and not</span> getattr(intake_record, field):
                setattr(intake_record, field, val)

    rag_context = <span class="str">"\n\n"</span>.join([f<span class="str">"[{r['source']}] {r['content']}"</span> <span class="kw">for</span> r <span class="kw">in</span> retrieved])

    <span class="cmt"># Step 3: Run the agent (tool calling + action plan generation)</span>
    agent_result = <span class="kw">await</span> run_intake_agent(intake_dict, rag_context)

    <span class="kw">return</span> {
        <span class="str">"intake_record"</span>: intake_dict,
        <span class="str">"case_id"</span>: agent_result[<span class="str">"case_id"</span>],
        <span class="str">"action_plan"</span>: agent_result[<span class="str">"action_plan"</span>],
        <span class="str">"resources"</span>: agent_result[<span class="str">"resources_found"</span>],
        <span class="str">"evidence"</span>: retrieved,                    <span class="cmt"># For transparency rail in UI</span>
        <span class="str">"tools_used"</span>: agent_result[<span class="str">"tool_calls_made"</span>],  <span class="cmt"># For audit display</span>
    }

      <h4>The offline angle is rare</h4>
      <p>Almost every other hackathon project assumes cloud. Make the offline-first nature central. Show the network indicator. Say "no data leaves this laptop" if there are healthcare judges.</p>
